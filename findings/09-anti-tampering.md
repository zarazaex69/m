# Finding: Anti-Tampering & Alt Client Blocking

## Подход к защите
Max использует **серверную** модель защиты, а не клиентскую.

## Что НЕ найдено (отсутствует):
- ❌ Root detection (нет проверок su, Magisk, Xposed)
- ❌ Frida/substrate detection
- ❌ Google Play Integrity / SafetyNet
- ❌ Emulator detection
- ❌ Debugger detection (anti-debug)
- ❌ Native integrity checks

## Что найдено:

### 1. APK Signature Verification
Класс `p000/szl.java` + `p000/cd5.java`:
```java
public Signature[] mo3530v(PackageManager pm, String pkg) {
    return pm.getPackageInfo(pkg, 64).signatures;
}
```
Проверяет подпись APK — отправляет на сервер при авторизации.
Сервер сравнивает с эталонной подписью VK.

### 2. Google Play Store Signature Check
Класс `p000/nu7.java`:
- Проверяет подпись `com.android.vending` (Play Store)
- Сравнивает с подписью `com.google.android.gms` (GMS)
- Определяет легитимность установки

### 3. CIS (Client Integrity Service)
- PmsKey: `cis-enabled`
- Серверный флаг — включает/выключает проверку целостности
- Вероятно отправляет fingerprint устройства + подпись APK на сервер
- Сервер решает блокировать или нет

### 4. VK Security Gate (упоминается в прессе)
В коде не найден как отдельный модуль — вероятно это:
- Серверная проверка подписи APK при каждом подключении
- Анализ поведения (аномалии в API вызовах)
- Проверка User-Agent и параметров устройства

## Как блокируются альтернативные клиенты

### Механизм:
1. При подключении клиент отправляет подпись APK
2. Сервер проверяет подпись против whitelist
3. Если подпись не совпадает → блокировка аккаунта
4. Дополнительно: `cis-enabled` может включать расширенные проверки

### Почему нет root detection:
- Max не пытается защитить данные на устройстве
- Цель — не дать использовать модифицированный клиент
- Серверная проверка подписи достаточна для этого
- Root сам по себе не мешает серверной верификации

## Обход (теоретический)
1. Подмена подписи через LSPosed/Xposed модуль
2. Патч smali для отправки оригинальной подписи
3. Использование оригинального APK с Frida hooks (подпись не меняется)

## Вывод
Защита минималистична на клиенте, но эффективна за счёт сервера.
Нет обфускации нативного кода, нет anti-debug, нет root detection.
Вся защита — проверка подписи APK на сервере + серверный бан.
