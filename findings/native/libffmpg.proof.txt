4. Configuration string is intentionally stripped
--------------------------------------------------
avcodec_configuration / avformat_configuration / avutil_configuration /
swscale_configuration  — all four functions return the same pointer
(adrp 0x59000 + 0x618 = 0x59618) which contains exactly one byte: 0x20 (' ').

A vanilla FFmpeg build embeds the full ./configure command (--enable-decoder=…,
--cc=…, --prefix=…) into FFMPEG_CONFIGURATION.  In libffmpg.so this define has
been replaced with a single space, hiding the actual build flags.

  → forensically meaningful: someone patched libavutil/utils.c (or the
    project Makefile) to scrub the configure line before compilation.

6. Diff between 26.15.3 and 26.16.0 builds of libffmpg.so
----------------------------------------------------------
$ cmp -l libffmpg-26.15.3.so libffmpg-26.16.0.so | wc -l
20

All 20 differing bytes lie in the .note.gnu.build-id section
(file offset 0x2e0 .. 0x2f3, 20 bytes = SHA-1 build-id).

The remaining 1 954 220 bytes (>99.999%) are byte-for-byte identical.

  → The native FFmpeg library was relinked but NOT rebuilt from updated
    sources between releases.  Six months passed (Nov 2022 → May 2026 since
    n4.4.3 was published) and no FFmpeg security backports have been
    incorporated into the shipping binary.
