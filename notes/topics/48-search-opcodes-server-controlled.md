# Поиск — WS-опкоды и server-controlled лимиты

## WS-опкоды поиска

| Опкод | Enum # | WS code | Что |
|---|---|---|---|
| `CONTACT_SEARCH` | 38 | 37 | поиск по контактам |
| `PUBLIC_SEARCH` | 56 | 60 | **публичный поиск** (каналы, боты, пользователи) |
| `CHAT_SEARCH` | 70 | 68 | поиск по чатам |
| `MSG_SEARCH_TOUCH` | 73 | 72 | тап на результат поиска по сообщениям (аналитика) |
| `MSG_SEARCH` | 74 | 73 | поиск по сообщениям |
| `CHAT_SEARCH_COMMON_PARTICIPANTS` | 134 | 198 | общие участники двух чатов |

## PmsKey поиска

| Ключ | # | Что |
|---|---|---|
| `pub-search-limit` | 291 | лимит результатов публичного поиска |
| `search-webapps-showcase` | (в rtd) | показывать мини-апки в результатах поиска |

`rtd.publicSearchResultsLimit` — accessor для `pub-search-limit`.

## Что важно

1. **`PUBLIC_SEARCH(60)`** — каждый поисковый запрос уходит на сервер по WS. Сервер видит все поисковые запросы пользователя (имена людей, каналов, ботов). Лимит результатов задаётся сервером через `pub-search-limit`.

2. **`MSG_SEARCH_TOUCH(72)`** — отдельный опкод для тапа на результат поиска по сообщениям. Это **аналитический опкод**: клиент сообщает серверу, на какой результат поиска пользователь нажал. Сервер знает не только что искали, но и что выбрали.

3. **`CHAT_SEARCH_COMMON_PARTICIPANTS(198)`** — поиск общих участников двух чатов. Это означает, что клиент может запросить у сервера «кто состоит и в чате A, и в чате B». Потенциально — инструмент для деанонимизации участников закрытых чатов.

4. **`search-webapps-showcase`** — мини-апки появляются в результатах поиска. Это означает, что поисковый запрос может триггерить показ конкретных мини-апок (server-controlled через PmsKey).

## Скептический разбор

- Серверный поиск — стандарт для мессенджеров. Telegram тоже отправляет запросы на сервер.
- `MSG_SEARCH_TOUCH` — аналитика кликов по результатам поиска. Это нормальная UX-аналитика, но в контексте государственного мессенджера означает, что сервер знает, что пользователь искал и что нашёл интересным.
- `CHAT_SEARCH_COMMON_PARTICIPANTS` — нетривиальная фича. Нужно проверить, доступна ли она обычным пользователям или только администраторам.

## Сводка

6 WS-опкодов для поиска. Все запросы уходят на сервер. `MSG_SEARCH_TOUCH` — аналитика выбора результата. `pub-search-limit` — server-controlled лимит. `CHAT_SEARCH_COMMON_PARTICIPANTS` — потенциальный инструмент деанонимизации.
