# Non-contact sync — серверно-управляемая синхронизация незнакомцев

В дополнение к [[06-contacts]] (двусторонняя синхронизация контактов из телефонной книги) — отдельный механизм для синхронизации **не-контактов**: пользователей, с которыми было взаимодействие, но которых нет в телефонной книге.

## PmsKey

| Ключ | # | Default | Что |
|---|---|---|---|
| `non-contact-sync-time` | (в rtd) | — | интервал синхронизации не-контактов (секунды) |
| `non-contact-max-chunk-size` | (в rtd) | 10 | максимальный размер батча при синхронизации |
| `non-contact-collection-interval` | (в rtd) | — | интервал сбора данных о не-контактах |
| `non-contact-complaints-enabled` | (в qp6) | — | включить жалобы на не-контактов |

## Механизм (myb.java)

`myb` — воркер синхронизации не-контактов. Логика:

1. Накапливает не-контакты в буфере (`ynb` — счётчик).
2. Когда буфер достигает `non-contact-max-chunk-size` (default 10) — триггерит синхронизацию.
3. Соблюдает интервал между синхронизациями (`non-contact-collection-interval`).
4. Использует `ReentrantLock` для thread-safety.

## Что важно

1. **Не-контакты** — это пользователи, которые написали пользователю или которым пользователь написал, но которых нет в телефонной книге. Сервер получает информацию об этих взаимодействиях через синхронизацию.

2. **`non-contact-max-chunk-size=10`** — сервер контролирует, сколько не-контактов отправляется за раз. При `0` — синхронизация отключена (`e()` возвращает `false`).

3. **Сервер знает граф взаимодействий** пользователя, включая людей, которых нет в его телефонной книге. Это расширяет [[06-contacts]] (телефонная книга) до полного графа общения.

4. **`non-contact-complaints-enabled`** — server-gated возможность жаловаться на незнакомцев. Если выключено — пользователь не может пожаловаться на незнакомца.

## Сводка

Отдельный воркер синхронизирует не-контакты (незнакомцев, с которыми было взаимодействие) с сервером батчами по `non-contact-max-chunk-size`. Сервер получает полный граф общения пользователя, включая людей вне телефонной книги.
