# Server-side struct: proxy / VPN / country / app-update — серверная диагностика клиента

`defpackage/ksg.java` — обработчик серверного сообщения, который парсит структуру с set параметров, относящихся к **серверной диагностике клиента** и server-pushed конфигу.

## Поля структуры

| Поле | Тип | Что |
|---|---|---|
| `proxy` | String | server-pushed proxy URL для outbound-трафика |
| `proxy-domains` | d60[] (массив строк) | список доменов, для которых использовать proxy |
| `isVpn` | bool | **серверная сторона детектит, использует ли клиент VPN** |
| `lang` | bool | флаг «язык подходит» / язык-detection passed |
| `callsSeed` | long | seed для генерации call-id |
| `reg-country-code` | d60[] | список разрешённых country code-ов для регистрации |
| `app-update-type` | int | enum типа update: 0=none, 1=optional, 2=force |
| `location` | String | server-detected country code пользователя (по IP) |

## Что важно

1. **`proxy` и `proxy-domains` пушатся с сервера**. Сервер может в любой момент пометить любые домены как «ходи через эту прокси» — что эквивалентно server-pushed выборочному прокси-файлу. Клиент применяет это в окружении HTTP-клиента.

2. **`isVpn` приходит от сервера**, а не определяется клиентом локально (хотя локально клиент тоже умеет — см. [[02-vpn-warning]]). То есть сервер делает свою независимую проверку (например, через TCP fingerprinting, BGP geolocation, или сравнение источников) и сообщает клиенту вердикт. Это важно: даже если пользователь переключит VPN после запуска MAX, серверная сторона пересмотрит свой `isVpn` при следующем хендшейке.

3. **`location`** — server-detected country code. То есть сервер знает, **в какой стране находится пользователь по IP**, отдельно от `reg-country-code` (где зарегистрирован телефон). Эти два значения могут отличаться (пользователь зарегистрирован с +7, но сидит в Германии) — клиент получает оба и может вывести соответствующий UI («вы не в РФ — фича X недоступна»).

4. **`reg-country-code`** — массив, не одиночное значение. Это значит, **сервер может разрешать пользователю регистрацию в определённой стране только если он там физически находится**. Или, наоборот, исключать страны.

5. **`app-update-type`** — серверный enum: 0=ok, 1=рекомендуется обновить, 2=force update (см. [[12-force-update-killswitch]]). Через эту структуру force-update имеет конкретное численное значение, плюс отдельный flow с redirect на `download.max.ru`.

6. **`callsSeed`** — параметр для предсказуемой генерации call-id'ов или подписи звонков. По имени — серверно задаваемый seed для PRNG в звонковом стеке. Если сервер контролирует seed — теоретически call-id'ы предсказуемы для серверной стороны.

## Сводка

Серверная сторона MAX не просто принимает запросы клиента, а активно **сообщает клиенту, что про него знает**: VPN включён или нет, в какой стране сидит, какие домены проксировать, какой regional-flow применить, нужно ли force-обновляться. Эта структура парсится клиентом при каждом подключении. PmsKey `proxy` и `proxy-domains` (см. [[03-pms-server-flags]]) — отдельный канал доставки той же информации.

Для пользователя, использующего VPN, это означает: **MAX сервер видит факт VPN независимо от вышибки `oneme_vpn_connected_*` в клиенте**, и может на основании этого менять поведение.
