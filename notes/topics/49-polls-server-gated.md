# Опросы (polls) — server-gated по типу чата и лимиту участников

Функция опросов в MAX полностью управляется сервером через 5 PmsKey.

## PmsKey опросов (из ezd.java с описаниями)

| Ключ | # | Тип | Описание из кода | Default |
|---|---|---|---|---|
| `render-polls` | 233 | bool | «Отображение опросов» | false |
| `polls-in-p2p-chats` | 234 | bool | «Создание опроса в диалоге» | false |
| `polls-in-p2g-chats` | 235 | int | «Лимит участников на создание опроса в чате» | 0 |
| `polls-in-channels` | 236 | int | «Лимит участников на создание опроса в канале» | 0 |
| `poll-ttl` | 237 | String (JSON) | «TTL поллинга опросов» | `{}` |

## WS-опкод

`GET_POLL_UPDATES(156, WS 306)` — получить обновления опроса (новые голоса).

## Что важно

1. **`render-polls=false` по умолчанию** — опросы выключены глобально. Сервер включает их через PmsKey-сегментацию.

2. **Гранулярный контроль по типу чата**: отдельные PmsKey для p2p-диалогов, групповых чатов и каналов. Сервер может разрешить опросы в каналах, но запретить в личных диалогах — или наоборот.

3. **`polls-in-p2g-chats` и `polls-in-channels` — int, не bool** — это лимит участников чата, при котором разрешено создавать опросы. Например, `polls-in-p2g-chats=100` означает «опросы доступны только в чатах до 100 участников». Default `0` = запрещено везде.

4. **`poll-ttl` — JSON-объект** — TTL (время жизни) опроса. Сервер задаёт, как долго опрос остаётся активным. Пустой JSON `{}` = default TTL.

5. **`GET_POLL_UPDATES(306)`** — клиент периодически запрашивает обновления голосов. Сервер знает, кто и когда проголосовал (это нормально для серверных опросов, но в контексте государственного мессенджера — это полный лог голосований пользователей).

## Скептический разбор

- Серверные опросы — стандарт. Telegram тоже хранит голоса на сервере.
- Что специфично: гранулярный server-gated контроль по типу чата и лимиту участников. Сервер может разрешить опросы только в «правильных» чатах.
- Анонимные опросы: неизвестно, поддерживает ли MAX анонимные опросы (как в Telegram). Если нет — сервер знает, кто как проголосовал.

## Сводка

5 PmsKey управляют опросами: глобальный switch, per-chat-type разрешения, лимиты участников, TTL. Все default=false/0 — опросы выключены по умолчанию. `GET_POLL_UPDATES(306)` — сервер получает все голоса.
