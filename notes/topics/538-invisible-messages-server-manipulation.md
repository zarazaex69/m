# 538. Invisible messages + серверная манипуляция сообщениями

## Суть

Сервер MAX имеет **полный односторонний контроль** над жизненным циклом сообщений на устройстве пользователя: может инжектировать невидимые сообщения, удалять любые сообщения без следа, модифицировать содержимое после доставки, и скрывать целые чаты.

## 1. Invisible Messages (серверная инъекция без UI)

WS-опкод `NOTIF_MESSAGE` (128) содержит поле `invisible` (boolean). Когда `invisible=true`:
- Сообщение сохраняется в SQLite (`d0c.X = true`)
- Уведомление пользователю **подавляется** (логика в `f0c.java:321`)
- Сообщение обрабатывается клиентом (может содержать команды, метаданные, payload)

**Импликация**: сервер может пушить произвольные данные на устройство, которые пользователь никогда не увидит.

## 2. Удалённое удаление сообщений (без следа)

### Через FCM push:
- `MessageRemoved` / `ChatMessageRemoved` — парсит `chatId`/`msgId`, удаляет из SQLite
- Работает даже когда приложение в фоне
- **Нет уведомления** пользователю об удалении

### Через WS:
- `NOTIF_MSG_DELETE` (142) — сервер пушит `chat_id` + `messageIds[]`
- Клиент выполняет: `UPDATE messages SET text = NULL, status = 10 WHERE id IN (...)`

### Soft-delete:
- Флаг `notDeleteMessageFromDb=true` — сообщение скрыто из UI, но **остаётся в БД**
- Потенциально доступно для серверного извлечения через heap dump или logcat

## 3. Модификация сообщений после доставки

Когда сервер пушит `NOTIF_MESSAGE` для уже существующего `messageId`:
- Клиент **перезаписывает** содержимое в БД
- Нет проверки, что редактирование пришло от оригинального отправителя
- Можно изменить: `text`, `elements`, `attaches`, `status`, `sender`, `time`

**Импликация**: сервер может подменить содержимое любого сообщения после доставки. Пользователь увидит изменённый текст.

## 4. Скрытие целых чатов

- `TYPE_CHAT_HIDE` / WS-опкод `CHAT_HIDE` (196) — чат исчезает из списка
- Может быть инициировано сервером

## 5. Ретроактивный TTL

- `TYPE_UPDATE_FIRE_TIME` (опкод 46) — сервер может задать время жизни **любому** уже доставленному сообщению
- Сообщение автоматически удалится по таймеру

## Полная таблица серверных возможностей

| Действие | Механизм | Видимость для пользователя |
|----------|----------|---------------------------|
| Инъекция невидимого сообщения | NOTIF_MESSAGE + invisible=true | Нет |
| Удаление сообщения | FCM MessageRemoved / WS NOTIF_MSG_DELETE | Нет |
| Модификация текста | NOTIF_MESSAGE на существующий msgId | Выглядит как редактирование |
| Скрытие чата | CHAT_HIDE | Чат исчезает |
| Автоудаление | UPDATE_FIRE_TIME | Сообщение пропадает |
| Soft-delete (скрыть, но сохранить) | notDeleteMessageFromDb=true | Скрыто, но в БД |

## Контекст

Без E2E шифрования сервер имеет plaintext всех сообщений в транзите. Эти механизмы дают ему контроль и над **клиентской копией** — может удалить улики, подменить доказательства, инжектировать фейковые сообщения.
