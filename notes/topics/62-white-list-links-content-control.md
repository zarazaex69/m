# white-list-links и show-warning-links — серверный контроль ссылок

## white-list-links (#314)

`PmsKey.f314whitelistlinks` — список доменов, которые считаются «безопасными» и открываются без предупреждения. Используется в `v7b.java:530` при обработке ссылок в сообщениях.

Логика: при клике на ссылку клиент извлекает hostname (`new URL(str).getHost()`), затем проверяет его против `white-list-links`. Если список пустой — ссылка считается «не в whitelist» и обрабатывается иначе (вероятно, показывается предупреждение или открывается через встроенный браузер).

Это означает: **сервер контролирует, какие домены считаются «доверенными»** в MAX. Сервер может добавить или убрать любой домен из whitelist без обновления клиента.

## show-warning-links

`PmsKey.f274showwarninglinks` — bool, default `false`. Описание из `e.java`: «show-warning-links». При `true` — показывать предупреждение перед открытием внешних ссылок.

Это server-gated включение предупреждений о внешних ссылках. Сервер может включить предупреждения для всех ссылок, не входящих в `white-list-links`.

## Что важно

1. **Сервер контролирует, какие сайты «безопасны»** для пользователей MAX. Это может использоваться для:
   - Блокировки ссылок на «нежелательные» ресурсы (VPN-сервисы, оппозиционные СМИ).
   - Принудительного открытия определённых сайтов через встроенный браузер (с возможностью инъекции заголовков).
   - Показа предупреждений перед переходом на «неодобренные» ресурсы.

2. **`white-list-links` — это не только безопасность, но и контент-политика**. Сервер решает, какие внешние ресурсы пользователь может открыть «без трений».

3. Связь с `show-vpn-snackbar` (PmsKey) — сервер уже показывает предупреждения о VPN. Аналогичный механизм для ссылок.

## Сводка

`white-list-links` — server-pushed список «доверенных» доменов. `show-warning-links` — server-gated предупреждения о внешних ссылках. Сервер контролирует, какие внешние ресурсы пользователь может открыть без предупреждения.
