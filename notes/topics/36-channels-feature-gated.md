# Channels — серверно-управляемый kill-switch фичи

Каналы (broadcast-фичи 1-N автору→подписчикам) — отдельная фича MAX, **по умолчанию выключенная серверной командой**. Wikipedia (август 2025): «функционал недоступен обычным пользователям».

## PmsKey, относящиеся к каналам

| Ключ | # | Что |
|---|---|---|
| `channels-enabled` | 76 | глобальный server-controlled gate включения каналов в клиенте |
| `channel-statistics-botid` | 192 | chat-id серверного бота «Статистика каналов» (для админов канала) |
| `channels-complaint-enabled` | 196 | разрешает жалобы на каналы |
| `channels-suggests-folder` | 230 | папка «Каналы» в боковом меню (server-pushed) |
| `channel-view-config` | 247 | конфиг режима просмотра канала |
| `create-channel-type-screen` | 174 | UI для создания канала (server-pushed) |
| `edit-channel-type-screen-enabled` | 175 | UI для редактирования |
| `bots-channel-adding` | 251 | разрешает ботам быть админами канала |
| `chat-history-warm-fail-interval` (косвенно) | — | прогрев истории канала |

WS-опкоды, относящиеся к каналам (см. [[20-ws-protocol-opcodes]]):

- `CHAT_LIVESTREAM_INFO(62)` — информация о live-streaming в канале
- `MSG_DELETE_RANGE(92)` — удаление диапазона сообщений (нужно админам канала)
- `CHAT_MEMBERS_UPDATE(77)` — обновление списка подписчиков
- `CHAT_PIN_SET_VISIBILITY(86)` — закрепление поста
- `CHAT_HIDE(196)` — скрытие канала из списка

## Что это значит

1. **Каналы — feature-flagged фича на сервере**. Если `channels-enabled = false`, UI и логика создания/просмотра каналов отключены даже если клиент с ними поставлен.
2. **Сервер может включить каналы для отдельных групп пользователей** (через `qtd-` сегментацию, см. [[03-pms-server-flags]]) — например, только для верифицированных аккаунтов или для пользователей в РФ.
3. **`channel-statistics-botid`** — отдельный server-bot, который владельцы каналов могут открыть для просмотра статистики подписчиков/просмотров. Это **серверная аналитика по каналу**, доступная клиенту только через диалог с этим ботом.
4. **Channel hide / Chat hide (`CHAT_HIDE(196)`)** — отдельный опкод. Может использоваться для **серверного скрытия канала**: пометить канал как «не показывать в этом регионе» или «у этого пользователя».
5. PmsKey `bots-channel-adding` — означает, что ботов можно делать **админами каналов**. То есть автоматизированный публикатор контента — встроенная штатная возможность.

## Скептический разбор

- Каналы как фича — норма для современного мессенджера (Telegram, Signal Stories, WhatsApp Channels). Ничего необычного в самом наличии.
- Что **специфично для MAX**: жёсткая server-gated роллаут (`channels-enabled` отключён по умолчанию в августе 2025), серверно-управляемая статистика канала через бот, и опкод `CHAT_HIDE` для скрытия канала.
- При централизованной модерации это означает: сервер может в любой момент **скрыть произвольный канал** у произвольного пользователя без удаления (тихая цензура). Это технически возможно по протоколу.

## Сводка

Каналы в MAX — server-gated роллаут (PmsKey `channels-enabled`). Управление статистикой канала — через серверного бота `channel-statistics-botid` (chat-id задаётся сервером, см. также [[11-state-bots-and-content-policy]] и [[22-gost-digitalid-family]] про шаблон system-bot). WS-протокол содержит механизм `CHAT_HIDE(196)` для серверного скрытия канала из списка пользователя — что архитектурно достаточно для тихой server-side модерации.
